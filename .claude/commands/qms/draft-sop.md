Draft a **Standard Operating Procedure (SOP/PR)** for: `$ARGUMENTS`

**SOP Structure:**

```text
Document Number: PR-<module>-<sequence>
Title: <Vietnamese title>
Version: 1.0
Status: Draft
Owner: <role>
Reviewer: <role>
Approver: CMMS Admin / Tổ HC-QLCL
Effective Date: <date>
Review Cycle: Annual
```

**Sections:**

1. **Mục đích (Purpose)** — why this procedure exists, what regulation it satisfies
2. **Phạm vi áp dụng (Scope)** — which assets, departments, roles this applies to
3. **Tài liệu tham chiếu (References)** — ISO clauses, regulations, related WIs
4. **Định nghĩa (Definitions)** — domain terms used in this procedure
5. **Quy trình (Procedure)** — numbered steps with responsible role for each
6. **Biểu mẫu (Forms)** — list of BM documents used in this procedure
7. **Hồ sơ lưu trữ (Records)** — what records are created, retention period
8. **Kiểm soát thay đổi (Change Control)** — how to revise this document
9. **Lịch sử thay đổi (Revision History)** — table of versions

**Regulatory Mapping:** Note which ISO 13485 clause and NĐ98 article this procedure satisfies.

**System Implementation:** Map each procedure step to the AssetCore workflow state or action that automates it.
