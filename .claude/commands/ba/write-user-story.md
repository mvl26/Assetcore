description: Write user stories for an AssetCore feature or module
---

Write **user stories** for: `$ARGUMENTS`

**Format:**

```text
US-XX | <Title>
As a <role>,
I want to <action/capability>,
So that <business value / clinical outcome>.

Priority: Critical / High / Medium / Low
Estimate: <story points>
Module: IMM-XX
```

**Acceptance Criteria (per story):**

3–5 bullet points — specific, measurable, testable. Use "must", not "should".

**Story Mapping:**

Group stories by user journey phase:

1. Navigate to feature
2. Input / create
3. Review / validate
4. Approve / submit
5. View result / audit

**Definition of Ready (before sprint):**

- [ ] Acceptance criteria written and agreed
- [ ] UI mockup or wireframe available (if UI story)
- [ ] Backend DocType and API endpoint identified
- [ ] Dependencies on other stories listed
- [ ] Regulatory basis noted (if compliance story)

**Roles Reference:**

CMMS Admin, Biomed Engineer, HTM Technician, Workshop Head, Tổ HC-QLCL, Clinical User
